Tier 1 — Auto-approve · confidence ≥ 90%
Claim status updated · settlement queued · customer notified (ACS email/SMS) with the plain-English “why”. Clears the routine ~60% of volume in minutes.
Tier 2 — Microsoft Teams review · 60–90%
Adaptive Card to the adjuster channel: evidence summary + one-click Approve / Adjust / Deny. The human decision is logged to the Glass Box like any agent’s.
Tier 3 — Escalate · < 60% or any flag
Senior adjuster + CSR with full transcript and audit trail. Injury, fraud signal, coverage dispute, customer distress → straight to a human.

End-to-end flow — the numbers on the wires
1 Customer signs in (Entra ID / mock SSO) — app hands policy context to Sara, who greets first
6 External MCP checks: NOAA weather, NHTSA recall, fraud / duplicate / estimate sandboxes
2 FNOL conversation: 911 triage → 4 questions → injury stop → CreateClaim service flow
7 Every agent logs WHY to Decision_Rationale before acting — the gold thread
3 Claim + first Glass Box row written in ~30 s (measured) — claim number back in chat
8 Theater Mode streams the audit feed live to the adjuster console (GetClaimAudit)
4 Row-added trigger fires MasterOrchestration — parallel fan-out to the swarm
9 Adjudication routes by confidence: ≥90 auto · 60–90 Teams card · <60 escalate
5 Extraction · Policy · Validation run concurrently on Azure AI services
10 Explanation Agent + ACS notify the customer; reopen / chase / status reuse the same audit thread
