Design rationale

Sara — the human touch

Real photo + named persona on the initiation screen + persistent header presence reduces the "I'm talking to a faceless system" anxiety. Pulsing green dot signals real-time. For demo: stock photo from randomuser.me. For prod: replace with a real claims-team photo or AI-generated avatar (HeyGen / D-ID can produce a 30-sec personalised greeting video as a Day-19 stretch).

Mandatory questions surfaced upfront

All required fields visible on screen 5 with red asterisks — no progressive disclosure that hides what's needed. Optional fields (other-driver info, witnesses) collapsed under a "+" toggle to keep the cognitive load low. Trust + speed.

Progress is honest

Step counter + time-remaining estimate ("~70 seconds left") set expectations. Anchors the "under 2 minutes" promise from the avatar greeting. Lets the user feel control.

Glass Box visible to the customer

Footer of the avatar card and the consent text on the review screen reference the audit trail. Customer-facing transparency is a real differentiator vs incumbents. Tappable link on the success screen would open a "why we asked X / what we checked" view (not in this mockup).

Empathy = moderate

"I'm really sorry you're going through this" once at the top, supportive microcopy at edges, "Talk to a person" persistent in header. Not over-cooked. State Farm-style trust + Lemonade-style warmth split.

Visual = trust-forward

Microsoft-blue primary (ties to the underlying stack), generous white space, soft shadows, rounded-2xl cards. Mobile-first 390×844 frame (iPhone 13/14 standard). Single accent — no rainbow gradients.

What's deliberately NOT in this mockup

  • Loss-type-specific branches beyond Collision (would clutter the spec preview)
  • Tier-3 escalation path (when the user is in distress)
  • Live-chat escape into Sara's actual chat window (placeholder button only)
  • Claim status/tracking deep-dive screens
  • Adjuster-side (that's Teams Adaptive Cards, not this surface)

React conversion notes

Each "screen" maps to one component (e.g. FNOLLogin.tsx, FNOLInitiate.tsx, FNOLQuestions.tsx). Shared header/avatar = SaraHeader.tsx. Form state = useReducer or react-hook-form. Submit = POST to a Power Automate HTTP-trigger flow URL → it creates Claim row → triggers orchestration. Total: ~6 components, ~400 lines for the working flow.
